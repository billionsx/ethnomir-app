## Критичные особенности реализации (из v2.0)

### Яркость скринов (баг ReportLab)
`c.clipPath()` накапливает graphics state. Решение в `pdfkit.py::draw_screen()`:
- clipPath убран; скругление через PIL (RGBA-маска)
- Скрин отдаётся как in-memory JPEG q=92 через ImageReader
- Обязательный `img.convert('RGB')` — убирает Google sRGB ICC

### Перенумерация (v2.1)
- `renumber.py` (если потребуется снова) обновляет `draw_page_frame` + `page_num=` + footer-цифры по PAGE_MAP
- Для cover-страниц hardcoded eyebrow/footer правятся ВРУЧНУЮ — в оригинале там были ошибки off-by-one
- Если добавляются новые страницы в середину — обновить PAGE_MAP и прогнать

### Паттерн новых страниц v2.1
- Eyebrow сверху (draw_eyebrow)
- Inter-Ex 34-36pt заголовок, Inter-Med 13-14pt подзаголовок
- Карточные сетки 2×4 / 3×3 / 2×2 / 2×3
- Стилизация через словарь `C` в pdfkit.py
- Все размеры протестированы, без перекрытий

### Плашки с динамической высотой
`box_h = paragraph_height + padding` вместо фиксированных 60pt.

## Стек и правила генерации

- Python 3.10+, ReportLab, Pillow
- Шрифт: Inter 400/500/600/700/800/900 в `assets/fonts/`
- Заголовки: Inter-Ex 34-36pt. Подзаголовки: Inter-Med 13-14pt. Текст: Inter 10-10.5pt leading 14-15.
- Палитра: словарь `C` в `pdfkit.py` — label, label2, label3, bg, sep, sep_light + цвета секций
- Для скринов всегда `draw_screen()`, никаких прямых `drawImage`

## Если нужны изменения

Стандартный workflow Billions X:
1. `git clone -b spravochnik-v2 ...`
2. Правим Python-файлы в `src/`
