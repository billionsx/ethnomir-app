## 3. Ключевые дизайн-решения

### Типографика
- **Inter** (6 весов 400/500/600/700/800/900) — основа. Загружена через `@fontsource/inter` (npm) → merge latin+cyrillic subsets → `.ttf`
- **DejaVu Sans** (Regular + Bold) — fallback для символов, которых нет в Inter: `₽`, `→`, `←`, `★`, `✓`, `©`, `™` и др. Зарегистрирован как `Sym` / `Sym-Bold`.

### Layouts модулей
- **60/40** — текст слева (58% ширины), один скрин справа. Используется для модулей с одним ключевым экраном (Главная, Жильё, Паспорт, Чеки, Чат, Франшиза).
- **gallery_2×2** — текст сверху полной ширины, 4 скрина сеткой 2×2 внизу. Для модулей с несколькими состояниями (Билеты, Услуги, Коллекции паспорта).
- **gallery_3×1** — 3 скрина в ряд внизу. Для модулей с 3 взаимосвязанными экранами (Этномир-хаб, Календарь/Сертификаты/B2B).

## 4. Решённые баги (чтобы не наступать повторно)

### 4.1 Кириллица в Inter без пунктуации
**Причина**: `@fontsource/inter` раздельно поставляет cyrillic и latin субсеты.
**Фикс**: `fontTools.merge.Merger().merge([latin, cyrillic])` для каждого веса. Результат — `.ttf` с кириллицей + латиницей + пунктуацией в одном файле.

### 4.2 Знак ₽ обрезался / исчезал
**Причина**: `₽ (U+20BD)` отсутствует в Inter (и в latin, и в cyrillic субсетах). При `c.drawString` он показывался пустым местом, при `Paragraph` — тоже.
**Фикс**:
- `draw_mixed(c, x, y, text, font, size)` — функция в `pdfkit.py`, которая разбивает строку на куски по шрифту и рисует каждый кусок через соответствующий шрифт (Inter или Sym). Используется вместо прямого `c.drawString` в подписях.
- `fix_special(text)` — оборачивает спецсимволы в `<font name='Sym'>₽</font>` для работы внутри `Paragraph`. Применяется автоматически через обёртку `Paragraph = lambda text, style: _RLParagraph(fix_special(text), style)`.

### 4.3 Бледные скрины
**Причина**: `drawImage(mask='auto')` с RGBA PNG применял альфа-канал как SMask, что визуально осветляло картинку при композиции.
**Фикс**: полностью убрали RGBA-маску. Теперь `draw_screen` использует `c.clipPath` — рукописный скруглённый path через `beginPath` + `arcTo`, а `drawImage` получает исходный PNG без альфы. Картинка полноцветная, углы скругляются на уровне PDF-клиппинга.

### 4.4 Подписи под скринами налезали друг на друга
**Причина**: `wrap_text_lines` не учитывал `\n` в строке — ломал весь текст на слова и склеивал. Отступы между ячейками и подписями были малы (8-10pt).
**Фикс**:
- `wrap_text_lines` теперь сначала разбивает по `\n` как жёсткий перенос, потом каждую hard_line — по ширине.
- Отступ подписи от скрина: 16-18pt (было 7-10).
- Межстрочный интервал подписи: 11pt.
- Подпись ограничена 2-3 строками через `[:2]` или `[:3]`.

### 4.5 Таблицы не переносили текст, вылезали за край
**Причина**: `Table(data, ...)` получала plain strings, ReportLab не переносил их.
**Фикс**: `ios_table()` теперь автоматически конвертирует все текстовые ячейки в `Paragraph` с соответствующим стилем — и текст переносится в пределах `colWidth`.

### 4.6 Скрины наезжали на заголовки
**Причина**: в layout 60/40 скрин стартовал с `PAGE_H - MARGIN_T - 70`, что при двухстрочном заголовке было слишком высоко.
**Фикс**: сдвинули всем скринам старт на `PAGE_H - MARGIN_T - 110`.

### 4.7 Ценностные пункты (01, 02, 03, 04) налезали друг на друга
**Причина**: номер `01` рисовался на той же `y`, что и заголовок (через `+10` offset), а потом `y -= phh + 2` — что давало слишком малый gap до следующего пункта.
**Фикс**: номер `c.drawString(cx, y - 10, num)` (в линию с baseline заголовка, но без offset), заголовок `ph_h.drawOn(cx, y - phh)`, между пунктами `y -= phb + 10` (было +5).

### 4.8 Quote-box перекрывал head
**Причина**: `Paragraph.drawOn(y - box_h + 12)` рисует снизу вверх, налезая на заголовок сверху.
**Фикс**: координата `y - 36 - p2h` (под заголовком + отступ, paragraph идёт вниз).

### 4.9 Водяные знаки «I», «II» на разделителях
**Причина**: 280pt `I` с цветом `#FFFFFF20` — reportlab не поддерживает прозрачность в `setFillColor` для текста, показывал сплошной жёлтый поверх синего.
**Фикс**: просто убрали водяные знаки. Разделители теперь чистые цветные страницы с заголовком и подзаголовком.

### 4.10 Footer стр. 37 — колонки перекрывались
**Причина**: левая колонка `BillionsX · Партнёр · billionsx.com` тянулась в одну строку через весь футер, правая `Этномир · Крупнейший парк` шла навстречу — в центре пересекались.
**Фикс**: чёткое деление на две колонки с multi-line текстом (название, роль, контакт/адрес в отдельных строках).

### 4.11 Скрин на стр. 4 и стр. 11 был одинаковым
**Причина**: я сначала везде использовал `01_13_45` (Этномир-хаб), потом по запросу заменил только в одном месте.
**Фикс окончательный**:
- **Стр. 4** (I слой, «Одно приложение. Весь парк.») = `01_06_56` — таб **Парк** (точка входа)
- **Стр. 11** (III слой, модуль M0 «Точка входа. Сегодня в парке.») = `01_13_45` — таб **Этномир** (Hero «Мир начинается с тебя»)

## 8. Принципы помощника при продолжении работы

1. **Не ломать то что работает** — код вёрстки хрупкий, каждое изменение требует визуальной проверки через `pdftoppm -r 70`. Пересобрал → растеризовал → посмотрел несколько страниц → только потом коммит.

2. **Сохранять единство дизайна** — заголовки 32pt `Inter-Ex`, body 10pt `Inter`, labels 8.5pt `Inter-Semi`. Eyebrow всегда UPPERCASE с `label2_real`. Цветовые маркеры ролей — строго согласно палитре.

3. **Использовать `draw_mixed` и `Paragraph`** — не писать `c.drawString(...)` напрямую для любых русских строк со спецсимволами. `₽`, `→`, `★` без fallback не отрисуются.

4. **Проверять через грид-превью** — `pdftoppm -png -r 70 file.pdf pg` + grid-скрипт из истории чата создаёт сетку 2×3 страниц. 7 сеток = полный проход по 37 страницам.

5. **Работать инкрементально** — добавил новую страницу → собрал → проверил → только потом следующая. Не писать сразу 10 страниц и собирать в конце.

6. **При правках существующей страницы** — находить конкретную функцию `page_XX()`, читать её целиком, править точечно. Не перевёрстывать с нуля.

## 9. Известные ограничения

- **Размер 8.3 МБ** — 69 PNG по ~130KB. ReportLab переиспользует изображения по пути — хорошо. Если понадобится меньше — через `pypdf.compress_content_streams()` или уменьшить PNG до 500×... (сейчас они в native разрешении 784×2024).
- **Время сборки** — ~4-5 секунд на машине средней мощности. Для итераций приемлемо.
- **Редактируемость** — все строки находятся в `sections_*.py` как Python-строки. Любая правка текста = правка `.py` → пересборка.
