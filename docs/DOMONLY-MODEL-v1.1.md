# ЗАКОН 1 · СТРАТЕГИЯ

## 1.1. Видение (10-летний горизонт)

К 2036 году каждая третья международная сделка по новостройке в мире проходит через Domonly. Платформа стала тем, чем Booking стал для отелей и Bloomberg — для финансовых рынков: невидимая инфраструктура, без которой отрасль уже не работает.

## 1.2. Главная стратегическая ставка

**Победа не в каталоге, а в инфраструктуре.**

Существующие игроки конкурируют на уровне "у кого больше объявлений" и "у кого красивее интерфейс". Domonly конкурирует на уровне "кто стандарт для интеграции застройщиков". Это другая лига — здесь побеждают сетевые эффекты, а не маркетинговый бюджет.

## 1.3. Структурная асимметрия рынка

Глобальный рынок продажи новостроек в 2026 году имеет три структурных дефекта, на которых строится возможность Domonly:

1. **Информационная асимметрия.** Покупатель из Москвы, ищущий квартиру в Дубае, не имеет доступа к качественной, верифицированной, актуальной информации. Локальные брокеры дают её фрагментарно и предвзято. Глобальные порталы дают каталог без сделочного слоя.

2. **Трансакционная асимметрия.** Сделка по новостройке между странами — это 7–10 контрагентов: брокер, застройщик, банк (ипотека), банк (FX), эскроу, юрист, налоговый консультант, переводчик, нотариус, ВНЖ-консультант. Каждый берёт свою маржу и риск-премию за непрозрачность.

3. **Технологическая асимметрия.** 95% мировых застройщиков управляют продажами через локальные CRM, локальные сайты, локальные отделы продаж. Их инвентарь не виден глобальному покупателю в едином формате.

Domonly закрывает все три асимметрии одной платформой.

### Логика matching

Победа Domonly — не в количестве рынков, а в количестве **активных коридоров** (source × hub). Каждый коридор — отдельная стратегическая единица с собственной юнит-экономикой, маркетинг-микс и compliance-стеком.

**Phase 1 (Год 1):** 1 hub × 3 sources = 3 коридора. Дубай × {RU, IN, CN}. ~250 сделок.

**Phase 2 (Год 2–3):** 5 hubs × 6 sources = 30 коридоров (выборочно). ~5 000 сделок.

**Phase 3 (Год 4–5):** 10 hubs × 10 sources = до 100 активных коридоров. ~30 000 сделок.

**Phase 4 (Год 6–7):** Открытая платформа, любой брокер/застройщик может подключиться через API. Domonly становится стандартом, как SWIFT для банков.

## 3.3. Layer 2 — Developer Layer (стандарт для застройщиков)

**Тезис:** через 5 лет каждый крупный застройщик мира должен иметь интеграцию с Domonly — как сегодня каждый отель имеет аккаунт в Booking.

### Что застройщик получает:

- **Глобальная видимость инвентаря** — без построения собственного международного маркетинга
- **Pre-qualified leads** — AI отсеивает нерелевантных, до застройщика доходит готовый
- **Реал-тайм аналитика** — дашборд по интересу: какой ЖК, какой планировке, из какой страны, в каком сегменте бюджета
- **Цифровая бронь** — слот резервирования через Domonly без human-touch
- **Эскроу-as-a-service** — деньги покупателя защищены, девелопер получает их по этапам

### Что застройщик платит:

- **Success fee** 1.5–3% от сделки (vs 5–7% классический брокер)
- **SaaS-подписка** $2–10K/месяц за интеграцию + аналитика (опционально)
- **Premium placement** — boost в результатах AI-консьержа (как Google Ads, но на инвентаре)

### Developer Integration Standard

Один раз интегрируется feed (REST + webhooks):
- `inventory.json` — статус по каждой квартире (free / reserved / sold)
- `pricing.json` — реал-тайм цены, спецпредложения
- `assets.json` — рендеры, планировки, документы
- `progress.json` — стадия стройки, фото, видео с площадки
- `legal.json` — пакет документов для покупателя

Это становится стандартом отрасли — конкурент с альтернативным форматом проигрывает.

## 3.4. Layer 3 — AI Agent Layer (мозг)

**Это главный технологический ров Domonly.**

### Архитектура (упрощённо):

```
User Query
    ↓
Domonly Concierge (LLM-фронт, dialog management)
    ↓
Intent Router (classification: search / question / book / advise)
    ↓
Specialist Agents (vertical experts):
  ├─ Match Agent          → ranking inventory
  ├─ Legal Agent          → cross-jurisdiction Q&A
  ├─ Tax Agent            → personal tax modeling
  ├─ Mortgage Agent       → cross-border financing
  ├─ Visa Agent           → residency by investment
  ├─ Negotiation Agent    → price/terms with seller
  └─ Document Agent       → drafting, review, KYC
    ↓
Action Layer (tool use):
  ├─ Reserve unit
  ├─ Schedule showing
  ├─ Draft contract
  ├─ Initiate escrow
  └─ Hand-off to JV human
```

### Ключевые агенты — что они делают:

- **Match Agent** учится на каждом клике, отказе, сделке. Через 18 месяцев ranks инвентарь лучше любого брокера.
- **Legal Agent** держит graph знаний: "новостройка в Турции для гражданина РФ" → 47 правил → ответ за 3 секунды.
- **Negotiation Agent** автоматизирует первый раунд переговоров: запрос скидки, проверка спецпредложений, бронь альтернативы — всё без человека на стороне Domonly.
- **Document Agent** генерирует draft договора в нужной юрисдикции, сравнивает с эталоном, помечает риски.

### Human-in-the-loop

AI делает 80%. На каждой сделке остаётся 20% человеческой работы — она передаётся **локальному партнёру JV**, не в собственный штат Domonly. Это сохраняет asset-light архитектуру.

Конкретные точки handoff:
1. Физический показ объекта (партнёр на земле)
2. Подпись финального договора (нотариус/юрист JV)
3. Любой conflict resolution (escalation путь)
4. Receipt of keys (партнёр сопровождает)

## 3.5. Layer 4 — Local JV Layer

**Юр. структура:** SPV (special purpose vehicle) в каждой hub-юрисдикции. Domonly — 51%, локальный партнёр — 49%.

### Что приносит локальный партнёр:

- Лицензии (broker license, real estate license, escrow agent)
- Знание рынка (specific developers, neighborhood, regulatory)
- Физическую сеть (агенты для показов, юристы, нотариусы)
- Локальный compliance (KYC, AML, tax reporting)
- Кризисный менеджмент

### Что приносит Domonly:

- Глобальный поток покупателей (CAC partner = 0)
- Технологическая платформа (CRM, AI, dashboards)
- Бренд + cross-border маркетинг

### JV Economics:

### Anti-free-rider механика:

# ЗАКОН 7 · AI

## 7.1. AI-стратегия Domonly

**Тезис:** Domonly — это AI-first компания, которая делает real estate. Не real estate-компания, которая использует AI.

Это означает: каждое продуктовое решение проверяется вопросом "может ли AI сделать это лучше человека?". Если да — AI делает, человек убирается. Если нет — это handoff в JV.

## 7.2. Архитектура: 4 уровня AI

### Уровень 1 — Conversational Layer (front)

LLM, скорее всего Claude Opus или собственный fine-tune. Главная функция — управление диалогом, понимание контекста, поддержание тона. Стоимость: оптимизация через caching и routing к более дешёвым моделям там, где не нужен top-tier.

### Уровень 2 — Specialist Agents (vertical experts)

Каждый агент — это обёртка вокруг LLM + специализированный knowledge graph + инструменты:

- **Match Agent** — собственный ranking-модель + LLM для justification
- **Legal Agent** — knowledge graph по 50 юрисдикциям + retrieval over юр. базы
- **Tax Agent** — модель + integration с tax APIs + LLM-объяснение
- **Mortgage Agent** — банковские партнёры + scoring + LLM-консьюлтация
- **Negotiation Agent** — обучен на исторических переговорах + game theory tools

### Уровень 3 — Action Layer (agentic)

Реальные действия в мире — через tool use:
- Создать резерв в инвентаре
- Запросить скидку у застройщика (auto + human approval)
- Назначить показ у JV-партнёра
- Сгенерировать draft контракта
- Запустить escrow setup
- Снять KYC

Это требует серьёзных guardrails и audit log. Каждое действие AI логируется и обратимо.

### Уровень 4 — Learning Layer (data flywheel)

Каждое взаимодействие генерирует данные:
- Что искал, что отверг, что выбрал
- На каком вопросе сорвался
- Какой ЖК показал лучшую конверсию для какого профиля
- Где переговоры зашли в тупик
- Какой документ требовал больше всего исправлений

Через 18–24 месяца это превращается в качественное преимущество, которое нельзя скопировать новым входом.

## 7.3. Data Network Effect — главный моат

Каждый новый покупатель тренирует AI лучше для следующего. Каждый новый застройщик расширяет покрытие. Каждый новый JV-партнёр углубляет локальное знание. Каждая закрытая сделка добавляет signal в ranking.

**Точка невозврата:** через 3 года с момента запуска, конкурент с теми же финансовыми ресурсами и тем же tech-стеком физически не может догнать Domonly, потому что у него нет 10 миллионов взаимодействий, на которых обучен наш AI.

Это классический моат Booking, не Compass.

## 7.4. AI-driven монетизация

AI напрямую генерирует revenue, не только снижает costs:

- **Price optimization** для застройщика: AI рекомендует динамическое изменение цен на основе спроса в реальном времени → revenue uplift 3–8% → Domonly берёт долю от uplift.
- **Dynamic placement:** AI решает, кому показывать какой ЖК → застройщики платят за лучший placement → premium revenue stream.
- **Negotiation automation:** AI добивается скидок у застройщика для покупателя → Domonly берёт долю от полученной скидки → win-win-win.

## 7.5. AI-этика и compliance

- **Explainability:** все рекомендации AI могут быть объяснены покупателю на простом языке
- **Bias mitigation:** регулярный аудит, что AI не дискриминирует по странам/языкам/бюджетам
- **Privacy:** покупательские данные не используются для тренировки моделей без opt-in
- **Auditability:** каждое решение AI в сделке логируется, доступно регулятору и покупателю
- **Human override:** покупатель в любой момент может затребовать живого консультанта

Это не косметика — это часть конкурентного преимущества в развитых рынках (EU, US), где AI-act и аналогичная регуляторика создаёт барьер для конкурентов с слабой compliance-готовностью.

- UX (интерфейс копируется за 6 месяцев)

Поэтому стратегия моата = data + integrations + compliance, а не "у нас крутой UI".

# РИСКИ И МИТИГАЦИЯ

| # | Риск | Вероятность | Импакт | Митигация |
|---|---|---|---|---|
| 1 | JV-партнёры начинают free-riding (используют наш бренд + трафик, обходят Domonly в сделках) | Средняя | Высокий | Контрактный exclusive lock + technological dependency: без эскроу Domonly сделка не закрывается |
| 2 | Регуляторика по странам бьёт по asymmetry (US RESPA, EU AI Act, CN capital controls) | Высокая | Средний | Phase 1 — только regulatorily friendly юрисдикции (UAE, PT, CY, TH); сложные рынки — позже, через локального compliance-офицера |
| 3 | AI-брокер не дотягивает до доверия в high-ticket сделках ($1M+) | Средняя | Средний | AI делает 80%, финальный handoff — всегда живой партнёр; для luxury — отдельный premium-flow с human-first |
| 4 | Capital intensity для запуска hub-а выше планового | Средняя | Высокий | Pilot 90-day с одним партнёром перед SPV; staged investment с KPI-trigger |
| 5 | Macroeconomic real estate cycle (как Китай 2022) | Средняя | Очень высокий | Diversification по hubs (разные экономические циклы) + сильный adjacent revenue (как у Beike, ремонт компенсировал падение sales) |
| 6 | Devastating compliance breach (FATF, OFAC) | Низкая | Catastrophic | Top-tier GC с дня 1; AML-стек сразу enterprise-grade; refusal-by-default для сомнительных сделок |
| 7 | Tech giant (Google, Meta) запускает конкурента | Низкая | Высокий | Speed to market + developer integrations as moat; + они не пойдут в regulatorymeшенный сегмент |
| 8 | Compass / Beike делают глобальный pivot | Очень низкая | Высокий | Их asset-heavy DNA не позволяет; они застряли в собственном рынке |
| 9 | AI-стоимость inference резко растёт | Средняя | Средний | Multi-model routing + caching + open-source fallback (Llama, Qwen) |
| 10 | Geopolitical fragmentation (RU sanctions, US-CN deals) | Высокая | Средний | Diversified source-markets; legal structure в нейтральных юрисдикциях (DIFC, Cayman) |

- vs **Beike**: Domonly cross-border, Beike — single-market
- vs **Самолёт Плюс**: Domonly asset-light + AI-first vs франшизная сеть
- vs **NoBroker**: Domonly marketplace с обеих сторон vs disintermediation
- vs **Compass**: Domonly без собственных агентов = фундаментально другая юнит-экономика
- vs **Opendoor / Loft**: Domonly без баланс-листового риска (мы не покупаем недвижимость)
- vs **Zillow**: Domonly — transaction-platform с дня 1, не portal
- vs **Square Yards**: Domonly с AI-моатом и cross-border сетью

# APPENDIX C — Глоссарий

- **Hub-city** — город/регион с высокой концентрацией спроса на новостройки от международных покупателей
- **Source-market** — страна, откуда едут международные покупатели
- **Corridor** — пара "source × hub" как отдельная стратегическая единица
- **JV-partner** — локальный брокер/девелопер, с которым Domonly строит SPV в hub-city
- **SPV** — special purpose vehicle, юр. лицо JV в локальной юрисдикции
- **Domonly Grid** — developer-side платформа: feed, CRM-mirror, аналитика
- **Domonly Bridge** — escrow + cross-border payments сервис
- **Domonly Lens** — verification & due diligence
- **Domonly Concierge** — premium human-augmented service tier
- **Domonly Network** — JV-партнёрская программа
- **Match Agent / Legal Agent / etc.** — vertical-specialist AI-агенты
- **Adjacent services** — сервисы вокруг сделки (ипотека, FX, ремонт, ВНЖ и т.д.), формирующие LTV
- **Decacorn** — компания с капитализацией $10B+
