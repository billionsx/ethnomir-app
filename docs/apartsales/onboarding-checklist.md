# О документе

Документ описывает, что Apartsales ожидает от застройщика-партнёра после подписания агентского договора, чтобы запустить онлайн-продажи лотов на платформе.

Структура — **6 волн**, от критичных к опциональным. Каждая волна снабжена дедлайном (от Day 0 — даты подписания договора). Без волн 1 и 2 запуск невозможен; волны 3–6 разворачиваются параллельно.

Со стороны Apartsales вся коммуникация идёт через **Key Account Manager** (контакт указан в шапке агентского договора), которого вы получаете на весь срок партнёрства.

# Кто такой Apartsales — коротко

Apartsales — продуктовая платформа онлайн-продаж первичной недвижимости. Полный цикл сделки проходит без офлайн-офисов: подбор → бронь → ипотека → ДДУ → электронная регистрация. Asset-light модель: мы не строим, не владеем складом квартир, не открываем шоу-румы. Мы — канал продаж нового поколения для застройщика.

Что мы даём партнёру:

- **Новый канал лидов** — целевой трафик с воронкой "от поиска до сделки" в одном интерфейсе.
- **Прозрачная аналитика** — read-only доступ к pipeline по вашим лотам в реальном времени.
- **Снятие нагрузки с отдела продаж** — наши менеджеры закрывают лиды, у вас остаётся только подтверждение брони и подписание ДДУ.
- **Интеграция в вашу шахматку, не наоборот** — ваша CRM остаётся источником истины. Apartsales работает поверх неё.

# Волна 2. Шахматка и техническая интеграция

**Дедлайн: Day 0 – Day 7. Критический путь.**

Без актуального inventory мы не платформа, а просто номер телефона. По убыванию предпочтения:

## 2.1. API (предпочтительный сценарий)

Если у вас уже подключены Profitbase / Macro / Sales Tools / amoCRM с feed-интеграцией — у вас стандартизированный API, и интеграция занимает 1–3 дня.

От вас потребуется:

- Документация API (или ссылка на провайдера)
- Тестовый и продакшн API-ключ
- Информация о rate limits
- Схема статусов лота: свободно / бронь / продано / архив (и любые промежуточные)
- Ответственный технический контакт со стороны провайдера или вашей IT-команды

## 2.2. Фид XML / JSON (запасной сценарий)

Если API нет, но есть фид (например, Cian / Авито / собственный):

- URL фида (с авторизацией, если требуется)
- Формат и схема полей
- Частота обновления (минимум 4 раза в сутки)

## 2.3. Excel-выгрузка (только как временное решение, до 30 дней)

- Регламент рассылки (минимум 2 раза в день в рабочие)
- Жёсткий формат столбцов (предоставим шаблон)
- Ответственный контакт за обновления

## 2.4. Минимальный набор данных по каждому лоту

Независимо от способа интеграции, по каждому лоту нам нужны:

- **Стабильный ID** (не меняющийся при ребрендинге, перезагрузках CRM и т.п.)
- **Локация**: корпус, секция, этаж, номер квартиры
- **Площади**: общая, жилая, кухня, балкон/лоджия (с коэффициентом)
- **Комнатность и тип**: студия / евро-1 / евро-2 / классика-1 / классика-2 / …
- **Отделка**: без отделки / white-box / чистовая / с мебелью
- **Ориентация и вид**: стороны света, на двор / улицу / парк / реку
- **Цена** (актуальная) и **цена за м²**
- **Статус** + срок брони (если в брони)
- **Ссылка на планировку** в SVG или PDF (без чужих водяных знаков)

# Волна 3. Коммерция

**Дедлайн: Day 3 – Day 10**

## 3.1. Прайс-политика

- Текущий прайс по всем лотам
- История цен помесячно за 12 месяцев (нужна для нашего pricing-движка и аналитики динамики)
- Регламент обновления прайса (как часто, кто инициирует)

## 3.2. Скидки и акции

- Условия 100%-оплаты
- Условия рассрочки от застройщика (срок, %, первый взнос)
- Сезонные акции с датами начала и окончания
- **Спец-условия для покупателей через Apartsales** — обсуждаемо как USP. Например, эксклюзивная скидка 1–3% или зафиксированная цена на 30 дней.

## 3.3. Бронирование

- Сумма брони (фиксированная сумма или % от стоимости лота)
- Срок брони
- Возвратная / невозвратная
- Возможность продления и условия
- Защита от double-booking (на чьей стороне)

## 3.4. Ипотечные программы

- Список аккредитованных банков с персональными менеджерами и контактами
- Спец-ставки от банков для конкретного ЖК
- **Субсидированная ипотека от застройщика** — есть/нет, ставка, срок
- Семейная ипотека — параметры
- IT-ипотека — параметры
- Льготная / государственная — параметры
- Trade-in — есть/нет, условия
- Материнский капитал — порядок применения
- Военная ипотека — есть/нет

## 3.5. Комиссия Apartsales

- Размер комиссии в % по типам сделок:
    - 100% оплата
    - Ипотека
    - Рассрочка от застройщика
    - Уступка прав
- Момент возникновения комиссии: бронь / ДДУ / гос.регистрация / раскрытие эскроу
- Срок выплаты с момента возникновения
- Документооборот: акт оказанных услуг + счёт + счёт-фактура (либо УПД)

# Волна 5. Процессы и команда

**Идёт параллельно волнам 1–4.**

## 5.1. Контакты со стороны застройщика — поимённо

Нужны имя, должность, телефон, email, Telegram:

- **Key Account Manager / Руководитель отдела продаж** — операционные вопросы, эскалации
- **Бронь-менеджер** — подтверждение броней, статусы лотов
- **Юрист** — согласование ДДУ, дополнительных соглашений
- **Ипотечный брокер / специалист** — сделки в банках
- **Маркетинг** — контент, согласование креативов
- **Финансы / бухгалтерия** — комиссии, акты, сверки

## 5.2. Каналы коммуникации

- Общий Telegram-чат: KAM Apartsales + ваша операционная команда
- Отдельный канал эскалаций для P0 / P1 ситуаций (срыв сроков, спорные брони)
- Email — для документооборота

## 5.3. SLA, на которые мы подписываем застройщика

| Событие | Норма |
|---------|-------|
| Подтверждение брони (рабочее время) | ≤ 30 минут |
| Реакция на лида с горячего касания | ≤ 15 минут |
| Подготовка ДДУ к подписанию | ≤ 2 рабочих дня |
| Обновление шахматки через API | real-time |
| Обновление шахматки через фид/Excel | ≥ 4 раза в день |
| Ответ на запрос по сделке от Apartsales | ≤ 4 часа в рабочее время |

## 5.4. Регламент сделки

- Схема подписания ДДУ: онлайн (УКЭП) / Госуслуги / удалённый нотариус / комбинация
- Подключение к электронной регистрации (Сбер ДомКлик / ВТБ / Росреестр напрямую)
- Регламент передачи документов после ДДУ покупателю
- Регламент раскрытия эскроу

# Волна 6. Аналитика и непрерывная работа

**Запускается с момента первой сделки.**

## 6.1. Что мы предоставляем застройщику

- **Еженедельный pipeline-отчёт** по нашим лидам в воронке: лид → квалификация → бронь → ДДУ → раскрытие эскроу
- **Дашборд в реальном времени** (read-only) с ключевыми метриками: CTR на карточку, конверсия в бронь, средний чек, скорость закрытия
- **Аналитика спроса** — какие планировки и корпуса активнее интересуют, что не смотрят

## 6.2. Что мы ожидаем от застройщика

- Read-only доступ в его CRM по нашим сделкам — или регулярный экспорт, если read-only невозможен
- Уведомления об изменениях (одним сообщением, без формальностей):
    - Смена цен
    - Запуск новых акций или окончание действующих
    - Открытие продаж новых корпусов
    - Изменения в ипотечных программах
- **Ежемесячная сверка по комиссиям** — фиксированный календарный день
- **Квартальный QBR (Quarterly Business Review)** — что продаётся, что нет, как улучшить конверсию, какие пулы лотов перевести в эксклюзив
