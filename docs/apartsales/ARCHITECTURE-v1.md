## 1. Видение

**Apartsales — это маркетплейс первичной недвижимости, где сделка совершается полностью онлайн без участия продажников.**

Аналогия из других рынков:

- **Aviasales** — мы убираем кассы авиакомпаний и турагентов. Билет покупается за 3 минуты с телефона.
- **Wildberries / Ozon / Amazon** — мы убираем продавца-консультанта. Корзина, чекаут, доставка.
- **Тинькофф** — мы убираем визит в офис банка. Ипотека одобряется и подписывается удалённо.

**Apartsales = Aviasales × Wildberries × Тинькофф для первичной недвижимости.**

Покупатель заходит на платформу, фильтрует юниты по любым параметрам, сравнивает по слоям аналитики (страна → регион → город → застройщик → ЖК → юнит), добавляет в корзину один или несколько объектов, нажимает «Купить», выбирает способ оплаты (полная / ипотека / эскроу / рассрочка / trade-in / маткапитал), подписывает ДДУ через Госключ и получает зарегистрированный документ из Росреестра в Госуслуги.

**Время от первого захода до подписанного ДДУ — целевое значение: 24 часа.**

## 2. Принципы

1. **Zero sales talk.** Покупатель никогда не разговаривает с менеджером по продажам застройщика. Если у него есть вопрос — он находит ответ в карточке юнита, в FAQ, в чате ИИ-ассистента, в видеоконсультации с независимым консультантом Apartsales (опционально, бесплатно). Менеджер застройщика подключается только на этапе подтверждения брони и подписания, и это технические шаги, не sales.

2. **Full digital.** Каждый шаг сделки — онлайн. Бумажный документооборот допускается только как edge-case (например, военная ипотека по требованию банка), и в этом случае Apartsales берёт логистику на себя.

3. **Полная прозрачность.** Цена в карточке юнита — финальная. Никаких «уточняйте у менеджера», «акция действует только сегодня», «звоните, расскажем». Все скидки, акции, ипотечные ставки, расчёт эскроу — открыто и сразу.

4. **Аналитика на каждом слое.** Каждый юнит, ЖК, застройщик, город, регион имеют публичный профиль с метриками: история цен, динамика спроса, скорость продаж, рейтинг, сравнение с рынком. Покупатель принимает решение на данных, не на эмоциях продажника.

5. **Asset-light.** Мы не строим, не владеем складом квартир, не открываем шоу-румы и шоппинг-моллы. Мы — слой между застройщиком и покупателем. Маржа — комиссия от застройщика за сделку.

6. **Concierge для застройщика.** Онбординг и операционка не должны напрягать застройщика. Мы делаем за него всё, что можем сделать. См. раздел 7.

7. **Стандартизация представления.** Каждый юнит, ЖК, застройщик упакованы в единый формат. Покупатель не должен переключаться между разными сайтами и форматами. Apartsales — единый стандарт качества контента и данных.

8. **Открытые API на выход.** Мы не запираем застройщиков. У них всегда есть полный экспорт своих лидов, броней, сделок, аналитики. Если они захотят уйти — могут уйти. Это снижает риск входа.

## 3. Архитектура продукта

Платформа состоит из трёх крупных слоёв:

```
┌────────────────────────────────────────────────────────────┐
│  Слой 1. Клиентский маркетплейс (B2C)                      │
│  apartsales.ru, iOS, Android                               │
│  Поиск, аналитика, корзина, чекаут, кабинет покупателя     │
└────────────────────────────────────────────────────────────┘
                            ↕
┌────────────────────────────────────────────────────────────┐
│  Слой 2. Backend-ядро и интеграции                         │
│  Inventory sync · Pricing engine · Mortgage routing        │
│  Document automation · Госключ · Росреестр · Эскроу        │
│  Banking · Госуслуги · Аналитический склад                 │
└────────────────────────────────────────────────────────────┘
                            ↕
┌────────────────────────────────────────────────────────────┐
│  Слой 3. Apartsales Lite — портал застройщика (B2B)        │
│  partner.apartsales.ru                                     │
│  Inventory · Лиды · Брони · Документы · Финансы · Аналитика│
└────────────────────────────────────────────────────────────┘
```

### 3.1. Клиентский маркетплейс (B2C)

**Каноничный путь покупателя:**

1. **Поиск.** Полнотекстовый, голосовой, фильтры (бюджет, комнатность, локация, метро, инфраструктура, срок сдачи, отделка, ипотечные программы, видовые характеристики, льготные категории).
2. **Карта и список.** Двойное представление с переключением. Кластеризация на карте, list-view с виртуализацией.
3. **Карточка ЖК.** Стандартизированная: рендеры, дрон, 3D-тур, фотохроника, генплан, инфраструктура, транспорт, школы/сады в радиусе, аналитика спроса по корпусам.
4. **Карточка юнита.** Планировка SVG (интерактивная), 3D-тур типового, цена, история цен, расчёт ипотеки, расчёт рассрочки, доступные банки, calculator total cost.
5. **Сравнение.** Таблица до 4-х юнитов одновременно, по 50+ параметров.
6. **Корзина.** Один или несколько юнитов (см. 5.1).
7. **Чекаут.** Способ оплаты, подача на ипотеку (если выбрана), подписание ДДУ через Госключ, открытие эскроу.
8. **Кабинет покупателя.** Статус сделки в реальном времени, документы, чат с поддержкой, история, рекомендации.

### 3.2. Apartsales Lite — портал застройщика (B2B)

**Это критический enabler масштабирования.** Без Apartsales Lite мы не сможем подключать средних и региональных застройщиков, у которых нет ни Profitbase, ни amoCRM, ни IT-команды.

**Главная идея:** Lite — это бесплатная мини-CRM для застройщика, замещающая отсутствующую инфраструктуру. По сути — мини-аналог Profitbase, который мы даём в подарок при подписании агентского договора.

**Что есть в Apartsales Lite:**

- **Шахматка.** Таблица всех юнитов с фильтрами по корпусам/секциям/этажам. Drag-and-drop статусы. Массовое обновление цен. История изменений.
- **Лиды.** Pipeline от Apartsales: новый → в работе → бронь → ДДУ → раскрытие эскроу. С каждой стадией прикреплены документы и комментарии.
- **Брони.** Активные брони, срок, статус, кнопка «подтвердить» / «отклонить» / «продлить». Push-уведомления в Telegram-бот.
- **Документы.** Сгенерированные ДДУ, акты, доп.соглашения. Скачивание PDF, отправка на подписание.
- **Финансы.** Комиссия Apartsales: начислено, к выплате, выплачено. Акты сверки автоматически. Счета и УПД генерируются нашей системой.
- **Аналитика.** Дашборд: лиды, конверсия, средний чек, скорость закрытия, динамика спроса, сравнение с рынком.
- **Команда.** Управление пользователями: бронь-менеджер, юрист, директор, бухгалтер. Роли и права.
- **Уведомления.** Telegram-бот для оперативки + email для документооборота.

**Как обновляется шахматка в Lite, если у застройщика нет API:**

1. Прямое редактирование в портале (вручную) — для самых маленьких.
2. Загрузка Excel один раз в день — наш парсер нормализует.
3. Telegram-бот: «Лот 12-204 продан» → бот меняет статус.
4. Email-парсинг: пересланное письмо с прайсом распознаётся.
5. Когда у застройщика появится API — переключаем на API без миграции данных.

**Lite — стандартная часть Concierge-тарифа** (см. раздел 7), бесплатна для застройщика.

### 3.3. Backend-ядро и интеграции

**Inventory Sync Engine.** Универсальный коннектор шахматок:

- API-адаптеры: Profitbase, Macro, Sales Tools, amoCRM, MAKVES, Bitrix24, 1С-Стройка.
- Feed-парсеры: ЦИАН, Авито, custom XML/JSON.
- Excel-importer (нормализация любого формата).
- Telegram-bot для ручных обновлений.
- Email-parser для рассылок прайсов.
- Output: единая каноническая модель юнита (см. раздел 4.1).

**Pricing Engine.** Анализ цен в моменте и динамики:

- Снимки цен каждый час по всем юнитам.
- Бенчмарки: ЖК vs ЖК в одном районе, юнит vs юнит в одном ЖК.
- Прогноз справедливой цены (ML).
- Подсветка «горячих» лотов и «зависших».
- Рекомендации застройщику по ценообразованию.

**Mortgage Routing.** Подача заявки на ипотеку:

- Сеть из 12+ банков-партнёров (Сбер, ВТБ, Альфа, Газпромбанк, Дом.РФ, Тинькофф, Совкомбанк, Росбанк, Открытие, Уралсиб, Промсвязьбанк, Райффайзен и т.д.).
- API-интеграции с банками, где доступно (Сбер ДомКлик API, ВТБ Открытый API).
- Форма-агрегатор: одна анкета → подача в N банков параллельно.
- Сравнение одобренных предложений в кабинете.
- Семейная, IT, Военная, Льготная, Maткапитал, субсидированная от застройщика.

**Document Automation.**

- Шаблоны ДДУ от Apartsales (юридически выверенные, актуализируемые).
- Авто-заполнение из данных юнита и покупателя.
- Версионирование шаблонов по застройщикам (если они настаивают на своём шаблоне).
- Сравнение редакций (если застройщик меняет шаблон — мы видим что именно).

**Госключ интеграция.** Подписание ДДУ:

- Покупатель: УКЭП через приложение Госключ.
- Застройщик: либо тоже Госключ (если есть УКЭП у директора/доверенного лица), либо через нашу инфраструктуру (доверенность + наш менеджер с УКЭП).
- Криптография: ГОСТ Р 34.10-2012 / ГОСТ Р 34.11-2012.

**Электронная регистрация в Росреестре.**

- Прямой канал в Росреестр через Сбер ДомКлик / ВТБ электронная регистрация.
- Альтернатива: прямая интеграция с Росреестром через ЕСИА (если получим аккредитацию).
- Оплата госпошлины — встроена в чекаут.

**Эскроу.**

- Открытие эскроу-счёта в банке, определённом проектной декларацией.
- Перевод денег покупателя на эскроу — встроен в чекаут.
- Отслеживание статуса до раскрытия (после ввода в эксплуатацию).

**Госуслуги интеграция.**

- Авторизация ЕСИА (для удобной идентификации покупателя).
- Доставка зарегистрированных документов в личный кабинет Госуслуг покупателя.
- Уведомления о статусе сделки в Госуслуги.

**Аналитический склад данных.**

- ClickHouse / Snowflake для аналитики.
- Все события платформы: просмотры, клики, сравнения, добавления в корзину, подачи, оплаты.
- Витрины для B2C (карточки), B2B (Lite-дашборд), внутренние (наш BI).

## 4. Аналитический слой

Это сердце продукта. Покупатель должен принимать решение на основе **публичных данных**, не «доверия к менеджеру».

### 4.1. Иерархия данных

Каждый объект имеет публичный профиль с метриками. Покупатель может «провалиться» с любого уровня вниз и подняться вверх:

```
Страна
  └── Регион (область / край / республика)
        └── Город
              └── Район города
                    └── Застройщик
                          └── Жилой комплекс (ЖК)
                                └── Корпус
                                      └── Секция
                                            └── Этаж
                                                  └── Юнит (квартира / апартаменты / коммерция)
```

Каждый уровень — это полноценная страница с аналитикой. Например:

- **Страна → Россия:** общее число юнитов в продаже, средние цены, топ-регионов, динамика.
- **Регион → Московская область:** число ЖК, средняя цена/м², прогноз сдачи в ближайшие 3 года.
- **Город → Калуга:** карта застройки, сравнение со среднеобластными значениями.
- **Застройщик → ПИК:** все ЖК, средняя надёжность сдачи (просрочки), кредитный рейтинг, история.
- **ЖК → «Зелёный квартал»:** генплан, динамика продаж, ипотечные программы, что осталось.
- **Юнит → Кв.157 в корпусе 3:** история цены, сколько раз в брони было, аналоги в этом и соседних ЖК.

### 4.2. Сравнение и фильтрация

**Фильтры на каждом слое.** Универсальный фильтр по 50+ параметрам:

- География (регион, город, район, метро, расстояние от точки)
- Цена (общая, за м², ежемесячный платёж по ипотеке)
- Площадь (общая, жилая, кухня)
- Комнатность и тип (студия, евро, классика)
- Этаж (избегать первый/последний и т.д.)
- Отделка (без / white-box / чистовая / с мебелью)
- Ориентация и вид
- Срок сдачи
- Класс ЖК (эконом / комфорт / бизнес / премиум / элит)
- Инфраструктура (школы, сады, парки в радиусе)
- Транспортная доступность
- Ипотечные программы (доступна семейная, IT и т.д.)
- Застройщик (надёжность, рейтинг)
- Эскроу-банк
- Особенности (двор без машин, кладовая, мастер-спальня, балкон/терраса, паркинг)

**Сравнение.** До 4-х объектов одновременно (юниты или ЖК), таблица параметров с подсветкой различий и преимуществ.

### 4.3. Дашбордность

**Личный дашборд покупателя:**

- Сохранённые поиски и алерты («появилась квартира до 10М в этом ЖК»).
- История просмотров.
- Корзина и список желаемого.
- Текущая сделка (если есть) — pipeline и документы.
- Калькулятор накоплений / ипотечной нагрузки.
- Рекомендации на основе поведения.

**Дашборд застройщика (в Lite):** см. 3.2.

**Публичные дашборды:** агрегированная аналитика рынка по городам и регионам (как Aviasales показывает «куда летают летом» — мы показываем «где растут цены», «где быстрее всего раскупают», «у каких застройщиков нет просрочек»).

## 5. Чекаут — революция воронки

### 5.1. Корзина из нескольких юнитов

**Зачем:** инвесторы покупают пулами (2–10 юнитов под аренду или перепродажу), семьи покупают «себе и родителям одновременно», корпоративные клиенты — для сотрудников. Сегодня это требует N независимых сделок и месяцев работы. У нас — одна сессия.

**Сложности и решения:**

- **Юниты у разных застройщиков** → корзина разделяется на под-сделки, каждая со своим ДДУ и эскроу. Покупатель видит единый чекаут, в бэкенде — N параллельных процессов.
- **Разные способы оплаты на разные юниты** → допускается. Один в ипотеку, другой за полную, третий в рассрочку.
- **Bulk-discount** → если у одного застройщика берётся 3+ юнита, автоматически применяется скидка по политике застройщика (если настроена в Lite).
- **Bulk-ипотека** → одна заявка в банк на несколько юнитов одного покупателя, банк решает индивидуально.
- **Сроки** → чекаут не блокируется самым медленным юнитом. Каждая под-сделка идёт своим темпом, отображается отдельный статус.

### 5.2. Способы оплаты

В чекауте покупатель видит для каждого юнита доступные способы:

1. **100% оплата** — перевод на эскроу одной транзакцией.
2. **Ипотека** — выбор банка из одобренных предложений (полученных через Mortgage Routing). Подача заявки и одобрение — внутри Apartsales.
3. **Рассрочка от застройщика** — если доступна по этому юниту/ЖК.
4. **Trade-in** — продажа имеющейся вторички в зачёт (через партнёра-агента + наш калькулятор).
5. **Материнский капитал** — заявка на использование, помощь с документами.
6. **Военная ипотека** — отдельный поток.
7. **Комбинации** — например, 30% маткапитал + 70% ипотека.

**Все расчёты прозрачны:** покупатель видит ежемесячный платёж, переплату, эффективную ставку, графики.

### 5.3. Подписание ДДУ через Госключ

**Поток:**

1. После выбора способа оплаты система генерирует ДДУ из шаблона + данных юнита и покупателя.
2. Покупатель проверяет PDF в кабинете, может задать вопрос юристу Apartsales (опционально).
3. Покупатель подписывает через приложение Госключ (УКЭП на смартфоне).
4. Застройщик подписывает: либо его директор/доверенное лицо через Госключ, либо менеджер Apartsales по доверенности.
5. Двусторонне подписанный ДДУ автоматически уходит в Росреестр на регистрацию.

**Если у покупателя нет Госключ** — встроенный onboarding: 5 минут на установку приложения, видео-инструкция, поддержка в чате. Если совсем не получается — возможна выездная команда нотариуса (платно, по желанию).

**Если у застройщика нет УКЭП** — три варианта в Lite:
1. Доверенность на Apartsales (мы подписываем за него).
2. Помощь с получением УКЭП (один раз, наш партнёр-УЦ).
3. Выездной нотариус для бумажного подписания (медленнее, но работает).

### 5.4. Электронная регистрация в Росреестре

- ДДУ + пакет документов + госпошлина уходят в Росреестр.
- Канал: Сбер ДомКлик API / ВТБ электронная регистрация / прямая интеграция с Росреестром (если получим).
- Срок регистрации: 3–7 рабочих дней (стандарт), 1–2 (через банк-партнёр).
- Покупатель видит статус в реальном времени.

### 5.5. Доставка документов в Госуслуги

- Зарегистрированный ДДУ автоматически попадает в личный кабинет Госуслуг покупателя.
- Уведомление push + email + Telegram (по выбору).
- Дубликат хранится в кабинете Apartsales бессрочно.

### 5.6. Что происходит дальше (post-purchase)

- Отслеживание стройки: фотохроника, веб-камеры, прогресс по графику.
- Уведомления о ключевых вехах (заливка, подкровельные, отделка, ввод в эксплуатацию).
- Раскрытие эскроу: автоматически при вводе.
- Приёмка квартиры: чек-лист, помощь приёмщика-партнёра (платно, по желанию).
- Передача ключей и регистрация права собственности — автоматически.
- Отзыв на застройщика и ЖК — публикуется на платформе (модерация).

## 6. Контент-производство в масштабе

Главная боль рынка — у каждого застройщика свой формат, свои файлы, своё качество. Покупатель видит хаос. Apartsales решает это **стандартизацией**.

### 6.1. Apartsales Studio

Внутренняя контент-команда:

- Фотограф + дрон-оператор.
- 3D-сканер (Matterport / Planoplan).
- Видеомонтажёр.
- Копирайтер (тексты по ЖК).
- Дизайнер (адаптация рендеров под наш стиль, чистка ватермарков, унификация планировок в SVG).

Один выезд — полный пакет контента под ЖК. Стоимость абсорбируется в комиссию первых сделок (для Concierge-тарифа) или продаётся как услуга (для Standard).

### 6.2. Стандартизация по уровням

**Юнит:** обязательный набор полей, единая SVG-планировка, единая 3D-тур-навигация, фото отделки в одном стиле.

**ЖК:** обязательный пакет — фасады с 4-х сторон, лобби, двор, генплан, дрон, промо-ролик, fly-through, инфраструктура, тексты по 4-м разделам.

**Застройщик:** профиль с фотохроникой всех текущих ЖК, метрики надёжности, история компании.

**Город / регион:** аналитические страницы с публичными графиками, формирующиеся автоматически из наших данных.

## 7. Операционная модель

### 7.1. Concierge by default

Базовый принцип: **онбординг — наша работа.** Если у застройщика чего-то нет — мы это делаем. Не передаём ему чек-лист, а сами закрываем.

### 7.2. Тарифы для застройщиков

Не пытаемся продавать onboarding-чек-лист. Предлагаем **выбор режима:**

| Параметр | **Standard (4%)** | **Concierge (6%)** |
|---|---|---|
| Интеграция шахматки | Их API / фид | Apartsales Lite (бесплатно) |
| Контент | Их рендеры/тексты | Apartsales Studio (включено) |
| ДДУ-шаблон | Их редакция | Наш шаблон + согласование |
| Ипотечные банки | Их сеть | Наша сеть из 12 банков |
| УКЭП / подписание | Их инфраструктура | Через Apartsales |
| Бронь-менеджер | Их сотрудник | На первое время — наш |
| SLA подтверждения брони | 30 мин | 4 часа (мягкий, первые 60 дней) |
| Маркетинг ЖК | Их команда | Наш контент-отдел |

Большинство региональных застройщиков в первый год выберут Concierge — и нам это нужно, чтобы расти.

### 7.3. Asset-light

Мы не строим, не владеем юнитами, не открываем шоу-румы. Наши затраты:

- Команда продукта и инжиниринга
- Apartsales Studio (амортизируется в комиссии)
- Маркетинг покупателю (B2C-трафик)
- Юристы и интеграции

Маржа = (комиссия от застройщика) − (cost-to-acquire покупателя + cost-to-onboard застройщика + операционка).

### 7.4. Onboarding застройщика — двухтрековая модель

(Подробно в `docs/apartsales/onboarding-checklist-v2.md` после переписывания).

**Track A — Express запуск (Day 0–2).** Минимум 6 пунктов: проектная декларация, разрешение на строительство, реквизиты эскроу, шахматка в любом виде, прайс, контакт бронь-менеджера. Лоты в эфире на третий день.

**Track B — Enrichment (Day 3–30, наша работа).** Контент, интеграция CRM, тренинг, юридическое полирование — мы достраиваем в фоне, не блокируя.

## 8. Технологический стек (предварительно)

> Стек выбирается в момент старта Phase 0. Ниже — текущая гипотеза, требующая утверждения.

**Frontend B2C:** Next.js 15 + React 19 + Tailwind CSS, развёртывание на Vercel или Cloudflare Pages.
**Frontend B2B (Lite):** тот же стек, отдельный домен `partner.apartsales.ru`.
**Mobile:** React Native (Expo) для iOS и Android, единая кодовая база с web где возможно.
**Backend:** Hono на Cloudflare Workers (edge-сеть, низкая латентность по РФ) либо классический Node/Go-монолит на VPS — решается на Phase 0.
**Database:** PostgreSQL (Supabase или self-hosted) для транзакционных данных, ClickHouse для аналитики, Redis для кэша.
**Поиск:** Meilisearch / Typesense / OpenSearch для full-text и геопоиска.
**Файлы и медиа:** S3-совместимое хранилище (Selectel / Yandex Cloud) + CDN (BunnyCDN / Cloudflare).
**Очереди и фоновые задачи:** Bullmq на Redis или Cloudflare Queues.
**Платежи:** ЮKassa / Тинькофф Эквайринг для онлайн-оплат, прямые банковские интеграции для эскроу.
**Идентификация:** ЕСИА (Госуслуги) как primary, email/SMS как fallback.
**Подписание:** Госключ SDK (или работа через УЦ-партнёра, если SDK недоступен).
**Госреестры:** API Росреестра (через банк-партнёра или напрямую при аккредитации).
**Мониторинг:** Sentry + Grafana + Prometheus.
**Аналитика поведения:** PostHog (self-hosted).

## 9. Дорожная карта

### Phase 0. Apartsales Lite (фундамент онбординга) — **первый приоритет**

**Логика:** Без Lite мы не можем подключать застройщиков без CRM. Это блокер Concierge-тарифа. Поэтому Lite строится первым.

**Объём Phase 0:**

- MVP-портал застройщика: шахматка (CRUD), лиды (Kanban), брони (подтверждение/отклонение), документы (генерация ДДУ из шаблона), уведомления (Telegram-bot + email).
- Inventory Sync Engine v0: ручной ввод, Excel-импорт, Telegram-bot для статусов.
- Минимальная B2C-витрина: список ЖК, список юнитов, карточки, форма заявки (без чекаута).
- Базовая аналитика для застройщика.
- Authentication, roles, audit log.

**Срок:** 3–4 месяца разработки.

**Критерий готовности:** один пилотный застройщик (без CRM) подключён, его шахматка живёт в Lite, первая бронь прошла полный цикл от лида до подтверждения.

### Phase 1. Onboarding-rewrite (двухтрековая модель)

**Зависимость:** Lite готов и работает.

**Что переписываем:**

- Внутренний playbook для KAM: «Как подключить застройщика за 48 часов».
- Шаблоны документов: договор бронирования (наш), агентский договор (наш), доп.соглашения, доверенности.

**Срок:** 2 недели после Phase 0.

### Phase 2. Sales deck для первой встречи

**Зависимость:** Phase 1 готова, у нас есть проверенный onboarding-flow.

**Что:**

- Pitch-deck «Apartsales за 10 минут» (PPTX/PDF, 10–12 слайдов).
- Скрипт KAM для первой встречи.
- Аудит-лист (20 вопросов, чтобы понять, в каком тарифе будет работать застройщик).

**Срок:** 2 недели после Phase 1.

### Phase 3 и далее. Достраивание маркетплейса

После того как у нас есть рабочая B2B-сторона и первые подключённые застройщики — приоритезируем B2C-фичи по реальному спросу:

- Полноценный чекаут с подачей на ипотеку (Mortgage Routing v1).
- Госключ-интеграция для подписания ДДУ.
- Электронная регистрация через Сбер ДомКлик.
- Корзина из нескольких юнитов.
- Аналитический слой по городам/регионам.
- Apartsales Studio как полноценный сервис.
- Mobile приложение (iOS, Android).
- Trade-in, маткапитал, военная ипотека.

Каждая итерация — новая партнёрская волна и расширение географии.

## 10. Риски и блокеры

**1. Юридическая допустимость.** Сделки с недвижимостью требуют квалифицированной электронной подписи и регистрации в Росреестре. Каждый шаг должен быть юридически валиден. Нужен профильный юрист с первого дня.

**2. Доверие банков.** Для интеграций с банками (особенно Сбер ДомКлик API) нужны партнёрские соглашения. Без них — ручные потоки.

**3. Аккредитация в Росреестре.** Прямой канал получить можно, но это процесс на 6–12 месяцев. До этого — через банки-партнёров.

**4. УКЭП покупателя.** Часть покупателей не имеет УКЭП и боится её получать. Нужен встроенный onboarding в Госключ, иначе теряем конверсию.

**5. Сопротивление застройщиков.** Старые игроки боятся прозрачности. Стратегия — заходить через средних и региональных, голодных до новых каналов. Топ-10 подключатся, когда увидят, что без нас отстают.

**6. Контент-затраты.** Apartsales Studio — большая операционная статья. Нужно жёстко считать unit-economics: контент окупается за N сделок.

**7. Регуляторные изменения.** ФЗ-214, ипотечное законодательство, эскроу-режим, льготные ипотеки — всё меняется часто. Нужен monitoring и быстрая адаптация.

**8. Кадры.** Маркетплейс недвижимости — узкая ниша, специалистов мало. Часть ролей придётся выращивать внутри.
