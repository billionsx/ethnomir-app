## Столп 2. Слоёная упаковка

**Тезис:** Мы продаём не юниты — мы продаём **6 слоёв ценности одновременно**: страна → регион → город → застройщик → ЖК → юнит.

На каждом слое — **отдельная маркетинговая упаковка**, своя landing-страница, свой контент, свои конверсионные элементы.

**Пример воронки:**

```
Иностранец гуглит «buy property russia»
   ↓
Landing «Russia for foreign buyers» (страна)
   ↓
Landing «Sochi: subtropical riviera» (регион / город)
   ↓
Landing «Top 10 developers in Sochi» (застройщик)
   ↓
Landing ЖК «Imperial Sochi» (комплекс)
   ↓
Карточка юнита #1407 (юнит)
   ↓
AI Concierge → KYC → бронь → сделка
```

**Архитектурное решение:** Headless CMS (#20) + автогенерация landing-страниц по шаблонам. Один шаблон — миллионы страниц на 6 языках.

## Столп 3. Двусторонний маркетплейс

### Стороны

**Покупатели (4 цели):**
1. **Для жизни** — семьи, переезд, ВНЖ, школы для детей.
2. **Для отдыха** — второй дом, курортная недвижимость, аренда сезонная.
3. **Для инвестиций** — yield, капитализация, токенизация (#21).
4. **Для работы** — релокация сотрудников, корпоративные пакеты.

**Поставщики:**
1. **Застройщики** — основной поставщик товара.
2. **УК** — управляют после сделки (#13).
3. **Собственники-инвесторы** — выход на ресейл через нас (#14).
4. **Государства** — экспортируют недвижимость через Apartsales Sovereign (#37).

### Динамика
- Двусторонний network effect (классика Aviasales/Booking).
- Reverse Marketplace (#22) — клиент описывает запрос, застройщики борются.
- В 80% сделок мы посредник, в 20% — принципал (через Apartsales Bank #24).

## Столп 4. Поиск и сопоставление

**Что:** Человек не должен фильтровать сам — ИИ должен матчить.

**Иерархия инструментов:**

| Уровень       | Инструмент                                | Кто использует            |
|---------------|-------------------------------------------|---------------------------|
| **L1 — basic**| Мегафильтр (200+ параметров)              | продвинутый пользователь  |
| **L2 — smart**| AI Concierge / Match Engine (#6)          | средний пользователь      |
| **L3 — auto** | Autonomous Buyer Agent (#25)              | премиум / ленивый         |
| **L4 — pre**  | Reverse Marketplace (#22)                 | целеустремлённый клиент   |
| **L5 — meta** | Apartsales Genome (#49) — психотип-матчинг| любой                     |

Каждый уровень — отдельный продукт, **все 5 живут одновременно**.

**Внутреннее ранжирование (#1 Apartsales Score™):**
- Юниты ранжируются по composite score: ликвидность × репутация застройщика × инфраструктура × юр. чистота × SLA-движок (#11).
- Низкий SLA-rating = автоматическое падение в выдаче. Защита клиента + давление на застройщика.

## Столп 5. Геймификация и удержание

### Геймификация клиента
- **Apartsales Points** (#16) — баллы за просмотры, бронь, KYC, сделку, отзыв, реферал.
- Уровни Bronze / Silver / Gold / Platinum / Black — открывают off-market сделки, скидки, сервисы.
- Достижения (collected всех ЖК города, обзорщик X стран, инвестор N сделок).
- Leaderboard инвесторов (опционально публичный).

### Геймификация застройщика
- **Developer Score** — внутренний рейтинг (SLA, качество фото, скорость ответа, NPS клиентов).
- Лиги: Bronze / Silver / Gold / Platinum.
- Awards «Застройщик года Apartsales» — медийное событие, ежегодное.
- **Геймификация менеджеров** внутри застройщика — leaderboard их сейлов, KPI, бонусы из подписки.

### Live Streaming Auctions (#29)
- Twitch-формат для эксклюзивных юнитов.
- Обратный аукцион с лимитом времени.
- Виральный канал для миллениалов и иностранцев.

## Столп 6. Стандартизация процессов

**Что:** Мы — самый удобный инструмент для застройщика. Чтобы это было правдой, мы стандартизируем всё.

**Стандарты:**
- **Онбординг застройщика** — 7 шагов, 1 день максимум, валидация юр. документов автоматическая.
- **Онбординг ЖК** — обязательная анкета (200+ полей), AI помогает заполнять.
- **Онбординг юнита** — 30 параметров minimum, фото 4K, 3D-тур (#7), документы.
- **SLA** — застройщик отвечает лиду за X часов, иначе наказание (#11).
- **Reporting** — застройщик получает разнослойные отчёты автоматически (см. `14-kpis-and-metrics.md`).
- **API-стандарт** — все CRM застройщика синхронятся через единый протокол (#9).

## Связь с реверс-моделью

Столпы 2, 3, 4 пересекаются в **Reverse Marketplace (#22) и Reverse Developer Model (#35)**:

- Столп 2 (упаковка) даёт нам слой контента.
- Столп 3 (двусторонний маркетплейс) даёт стороны.
- Столп 4 (поиск/матчинг) переворачивает поток: не клиент ищет, а застройщик находит спрос.

Это и есть **категориеобразующая фишка**, которая делает нас не «улучшенным ДомКликом», а новой категорией.
