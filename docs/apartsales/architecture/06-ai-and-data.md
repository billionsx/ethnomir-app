# 06. AI и Data слой

ИИ — не feature, а **операционная система продукта**. Все 9 продуктов этого файла связаны через единое data lake — Apartsales Data.

---

## #6 — AI Concierge / Match Engine

**Что:** Главный интерфейс продукта. Не фильтр — диалог.

**UX:**
```
Ты: «Семья 4 человека, 25 млн, Сочи, для жизни, въезд через 18 мес.»
ИИ: «Под твой запрос подходит 7 ЖК. Топ-3 с обоснованием:
     1. Imperial Sochi — Score 87, AS-Index +18% за 12 мес,
        школа в шаговой доступности, сдача через 16 мес.
     2. ...
Хочешь, я задам уточняющие вопросы или забронирую тур по Digital Twin?»
```

**Стек:**
- Базовая модель: Claude Sonnet / Opus + GPT-5 (multi-provider redundancy).
- RAG: vector search по нашему data lake — все юниты + транзакции + reviews.
- Tool calling: бронирование, KYC, mortgage application — всё прямо из чата.
- 6 языков нативно.

**Зачем:** UX-моат. Чем больше диалогов — тем умнее. Конкуренту нужно 2–3 года на копирование наших данных.

**Метрика:** 60%+ всех сделок инициированы через Concierge к Y3.

## #7 — Digital Twin юнита

**Что:** Полная цифровая копия каждого юнита.

**Состав:**
- 3D-тур (Matterport / Pano2VR) или Apartsales-собственный сканер.
- AR-приложение — поставить юнит в свою комнату через iOS/Android.
- Drone-видео ЖК и района.
- Полный пакет документов в одном месте.
- История изменений (планировка, цена, владельцы — для ресейла).
- Quantum Pricing коридор (#48).

**UX:** Иностранец из Манилы «проходит» по квартире в Сочи без билета.

**Стандарт:** Twin обязателен для попадания в топ-выдачу (часть Apartsales Score™).

## #8 — Apartsales Data

**Что:** Анонимизированная аналитика рынка как продукт.

**Подписчики:**
- Банки — для скоринга залогов.
- ESG-фонды — для оценки портфеля.
- Девелоперы-конкуренты — для рыночной разведки.
- Госорганы — для мониторинга рынка.
- Хедж-фонды — для торговли AS-Index деривативами.
- Журналисты — для отчётов.

**Тарифы:**
- Free tier — отчёты топ-50 городов помесячно.
- Pro $499/мес — все города + API.
- Enterprise $9999/мес — raw data + custom queries.
- Government tier — спец-условия.

**Зачем:**
- Дополнительный revenue stream (Y3+: $50–200M ARR).
- Усиление моата (чем больше данных, тем ценнее, тем сложнее догнать).
- PR через цитируемость в Forbes/РБК/Bloomberg.

## #23 — Apartsales Index (AS-Index)

**Что:** Биржевой индекс мировой первички. Котируется ежедневно.

**Структура:**
- AS-Global — глобальный композит.
- AS-Russia, AS-UAE, AS-Turkey — страновые.
- AS-Sochi, AS-Dubai, AS-Istanbul — городские.
- AS-Premium / AS-Mid / AS-Mass — сегментные.

**Методология:** Hedonic regression на наших транзакционных данных + corrections from listings (по примеру S&P/Case-Shiller).

**Roadmap:**
- **Y1:** AS-Russia + AS-Sochi публикуем еженедельно, открыто, бесплатно.
- **Y2:** ежедневно + 10 индексов + платный pro-tier.
- **Y2.5:** деривативы — форварды и опционы на AS-Index (через партнёрство с Мосбиржей / DGCX).
- **Y3:** ETF на AS-Russia (через российского управляющего).
- **Y5:** ETF mass-market в США/ЕС через partner-фонд.

**Зачем:** Превращает Apartsales из маркетплейса в **финансовую инфраструктуру**. Bloomberg первички. Forbes/РБК будут цитировать AS-Index ежедневно — бесплатный медиаканал на десятилетия.

## #25 — Autonomous Buyer Agent

**Что:** AI-агент, которому ты делегируешь поиск и покупку.

**UX:**
```
Ты: «Найди мне квартиру в Дубае 2BR, до $500K, готовая к сдаче,
     yield не ниже 7%, район walkable. Бюджет на покупку — 30 дней.»
Агент: «Принято. Я начинаю мониторинг и буду писать только при значимых событиях.»

[через 4 дня]
Агент: «Нашёл 3 кандидата. Один — на 5% ниже рынка из-за срочности.
        Я начал торг через Reverse Marketplace, продавец готов сдвинуть до –8%.
        Подтверждаешь продолжение торга?»
```

**Возможности:**
- Мониторинг 24/7.
- Автоматический торг через Reverse Marketplace (#22).
- Подготовка документов / KYC / ипотеки.
- Биометрическое подтверждение для финального шага.

**Зачем:** Категория, которую невозможно догнать существующими CRM. Concierge (#6) консультирует — Agent **действует**.

**Tier:** Премиум-фича для P2 / P5 / P42.

## #26 — Dynamic Pricing Engine для застройщиков

**Что:** Динамическое ценообразование как у авиакомпаний.

**Факторы модели:**
- Спрос (просмотры / брони / заявки на ипотеку за неделю).
- Срок до сдачи.
- Конверсия воронки.
- Локальный AS-Index.
- Конкурирующие ЖК.
- Сезонность.
- Геополитические триггеры.

**UX для застройщика:**
- Модуль в их CRM показывает: «Снизь юнит #1407 на 2% — конверсия вырастет 18%, чистая выручка +9%.»
- Auto-mode — система сама меняет цены в заданном коридоре.

**Юнит-экономика:** +10–20% к выручке застройщика через 6 мес после внедрения.

**Через 2–3 года это станет фактическим стандартом ценообразования в первичке.**

## #38 — Predictive Hype Engine

**Что:** ИИ предсказывает, **где будет хайп через 3–7 лет**, до того как это очевидно.

**Источники данных:**
- Миграционные потоки (passport stamps, Telegram-каналы релокантов).
- Инфраструктурные проекты (госконтракты, тендеры).
- Изменения в законодательстве (ВНЖ, налоги).
- Климатические модели (30-летние прогнозы).
- Telecom traffic patterns.
- Ночные снимки со спутников (Planet, Maxar).
- Поисковые тренды по геолокациям.
- Банковские транзакции (агрегированные через open banking).

**UX:** Pro-инвестор открывает терминал, видит «горячие зоны через 5 лет» с confidence scores и драйверами. «Сочи 2018» в 2024 году.

**Revenue:** Подписка $499–9999/мес — это новый Bloomberg Terminal для недвижимости.

## #40 — Apartsales Time Machine

**Что:** Полная история любого юнита — кто строил, кто покупал-продавал, динамика цены, изменения соседей, события района, прогноз на 10 лет вперёд.

**UX:** Ползунок времени на странице юнита. Видишь всю историю + прогноз.

**Зачем:** Прозрачность убивает мошенничество. Иностранец видит: «продавался уже 4 раза, средний срок владения 2.7 года, цена росла на 12% в год — нормально для этого типа объекта».

**Источники:**
- Наши транзакционные данные.
- Партнёрство с государственными реестрами (Росреестр, DLD, etc.).
- Apartsales Score™ + AS-Index исторические значения.

## #48 — Quantum Pricing Engine

**Что:** Симулятор сценариев на каждый юнит.

**Что показывает:** Распределение возможных цен через 1/3/5/10 лет по 10000+ симуляций Монте-Карло.

**Учитываемые факторы:**
- Ставка ЦБ ±N%.
- Инфляция.
- Геополитические сценарии (санкции, отмена санкций, войны).
- Климатические сценарии.
- Олимпиада / EXPO / другие мега-события.
- Демография района.
- Развитие транспортной инфраструктуры.

**UX:** «Цена сейчас 12М ₽. Через 5 лет: медиана 18М, 90% коридор 14–24М, 99% коридор 11–32М. Главные драйверы downside: ставка ЦБ. Upside: метро через 4 года.»

**Это McKinsey + Goldman Sachs-уровень аналитики, упакованный для розницы.**

## #49 — Apartsales Genome

**Что:** Биометрический и поведенческий профиль клиента → матчинг по психотипу.

**Источники:**
- Поведение на платформе (на чем задержался, что сравнивал).
- Опционально: подключение Apple Health / Google Fit (ритм жизни).
- Опционально: тест из 50 вопросов (тип Big Five + ценности + lifestyle).
- Опционально: интеграция с банком (тип трат → lifestyle).

**Что выдаёт:**
- «В этом ЖК 67% жильцов с твоим психотипом.»
- «Этот район — для интровертов; ты экстраверт, посмотри вот эти 3.»
- «Среди успешных в твоей когорте популярны эти 12 объектов.»

**Зачем:** Революция в принципе подбора. Не «3 комнаты, 5 этаж, центр», а **«среда, где ты будешь счастлив».**

**Никто в мире этого не делает.**

## #50 — Real Estate Prediction Markets

**Что:** Polymarket для недвижимости.

**Примеры рынков:**
- «AS-Sochi вырастет на 20%+ к концу 2026?» — yes/no.
- «ЖК Imperial Sochi сдадут до Q3 2027?»
- «Застройщик X объявит банкротство до конца года?»
- «Ставка ЦБ опустится ниже 10% до конца 2026?»

**Зачем:**
- Аналитика толпы — мудрость рынка как бесплатный сигнал для AI Concierge.
- Виральный продукт для геймеров и крипто-аудитории.
- Хедж для серьёзных инвесторов (страхование позиций).
- Дополнительный revenue (комиссия 1–2% с оборота).

**Юрисдикция:** Через лицензированные prediction markets (Kalshi, Polymarket-партнёрство), либо собственная лицензия в DIFC / VARA.

## Архитектура data lake

Все 9 продуктов работают с одним data lake:

```
Apartsales Data Lake
├── Transactions (anonymized)        ← база для AS-Index, Score™
├── Listings & changes               ← база для Pricing, Hype
├── User behavior                    ← база для Concierge, Genome
├── External signals                 ← база для Hype, Quantum
│   ├── Satellite imagery
│   ├── Mobility data
│   ├── Search trends
│   └── Government registries
├── ESG data                         ← база для #36
└── Predictions ledger               ← база для Time Machine
```

**Стек:**
- ClickHouse для транзакций / time-series.
- Pinecone / Weaviate для векторного поиска (Concierge).
- BigQuery / Snowflake для analytical workloads.
- Kafka для real-time signal ingestion.
- dbt + Airflow для transformations.
- MLflow для model registry.
