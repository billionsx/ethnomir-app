## Что это за документ

`pdf/ethnomir_spravochnik_v2.pdf` — полноценный 37-страничный справочник, который в одном месте отвечает на основные вопросы о цифровом продукте ethnomir.app:

- **Что это**: цифровая операционная система парка, объединяющая 15 отдельных IT-систем (касса, PMS отелей, POS ресторанов, CRM, лояльность, маркетплейс, инвест-лендинг, франчайзинговый портал, 22 лендинга)

- **Куда движется**: роадмап Q2-Q4 2026 по трём направлениям (потребительский опыт, интеллект/персонализация, масштабирование)

## Для каких задач использовать

### 1. Презентация руководству
Показать объём проделанной работы и текущее состояние продукта. Документ структурирован как McKinsey-отчёт: от общего к частному, от «одним взглядом» до инфраструктурных деталей.

### 2. Онбординг новых участников
Новый инвестор, партнёр, ключевой сотрудник — один документ даёт понимание всего продукта за 30-40 минут чтения.

### 3. Аудит и передача прав
На стр. 37 собраны все технические идентификаторы (GitHub, Vercel, Supabase project IDs, авторизация, платежи). В случае передачи прав или смены подрядчика — это готовый артефакт передачи.

### 4. Инструмент продаж и переговоров
Раздел V «Ценностные карты» для каждой аудитории — готовая аргументация при переговорах с партнёрами-ресторанами, потенциальными франчайзи, инвесторами в недвижимость Посёлка Мир.

### 5. Коммуникация с франчайзи
Раздел III модуль 08 + раздел V карта «Для франчайзи» дают полное понимание того, что получает франчайзи в рамках платформы — без необходимости отдельной демо.

## Что НЕ входит в этот документ (и почему)

- ❌ **Коммерческое предложение** / цена разработки / ROI инвестиций в разработку — для этого отдельный коммерческий документ. В v2.0 справочник — pure product reference, без цен.
- ❌ **Маркетинговые материалы** / брошюры для гостей — это product sheet, не маркетинг
- ❌ **Юридические документы** / соглашения / NDA — упомянуты как часть системы, но в отдельных файлах
- ❌ **Операционные инструкции** / «как пользоваться CRM» — это справочник о системе, не учебник

## Как читать

**Если 5 минут**: стр. 1-5 (обложка, оглавление, продукт в одном взгляде). Получишь общую картину.

**Если 15 минут**: добавь стр. 25-30 (CRM — операционная система). Увидишь что управление парком действительно централизовано.

**Если час**: прочти все 37 страниц по порядку. Запомнится надолго.

**Если нужна конкретика по модулю**: оглавление (стр. 2) содержит точные диапазоны страниц по каждому слою.

## Обновление документа

Когда нужно будет обновить цифры и/или добавить новые скрины:

1. Снять свежие метрики из Supabase и GitHub (инструкция — в `docs/PROJECT-NOTES.md`, раздел 6)
2. Обновить константы в файлах `src/sections_*.py`
3. Добавить новые скрины в `screens/` (если нужно)
4. Запустить `python3 src/build_full.py`
5. Новый PDF — в `pdf/ethnomir_spravochnik_v2.pdf`

Работа с документом возможна из любого AI-чата — всё необходимое (код, скрины, шрифты, каталоги) хранится в этом репозитории.
